# Проект БД — рабочий репозиторий

Командный проект по курсу «Конструирование баз данных». Цель — один итоговый документ (`Итоговый документ.md`) + рабочие SQL-файлы.

## Структура

| Файл | Что внутри | Владелец |
|---|---|---|
| `Итоговый документ.md` | Сдаваемый документ из 16 разделов — единый источник правды | Роль 4 собирает, пишут все |
| `schema.sql` | Каноническая схема: DDL, ключи, ограничения, индексы | Роль 1 + Роль 3 |
| `dml.sql` | Запросы (CRUD + сложные) и транзакции | Роль 3 |
| `seed.sql` | Тестовые данные (5 клиентов, 3 ресторана, 9 заказов…) — проверены на PG 18 | Роль 3 |
| `schema.dbml` | Исходник ER-диаграммы для dbdiagram.io (вставить → картинка) — проверен | Роль 1 |
| `ai_log.md` | Лог диалогов с AI по ходу проекта | пополняют все, курирует Роль 4 |
| `decisions.md` | Журнал спорных проектных решений | пополняют все, оформляет Роль 4 |
| `docker-compose.yml` | Воспроизводимое окружение: PostgreSQL 18 + проектные SQL внутри контейнера | Роль 3 |
| `TODO.md` | Финальный статус готовности и ручные действия перед сдачей | вся команда |

## Роли

1. **Архитектор схемы** — ER, каноническая схема/DDL, консистентность, мерж изменений.
2. **Аналитик зависимостей и нормализации** — текстовые ограничения, FD/MVD, нормализация, аномалии.
3. **SQL-разработчик** — реализация DDL, DML, транзакции, нефункциональные требования/индексы.
4. **AI-координатор и редактор** — раздел про AI, верификация 2-й LLM, спорные решения, сборка документа.

## Правила работы

- **Единый источник правды:** схема живёт только в `schema.sql`. Копий не плодим.
- **Schema freeze** после фазы нормализации — схему меняет только Роль 1.
- **Пишем по ходу:** каждый удачный диалог с AI → `ai_log.md`; каждый «здесь был выбор» → `decisions.md`. Не оставляем на конец — реконструкция задним числом видна и режет баллы.
- **Домен:** доставка еды (маркетплейс ресторанов).

## Окружение

Вариант A — Docker (рекомендуется, версия PG фиксирована):

```bash
docker compose up -d
docker compose exec db psql -U perekus -d perekus -f /project/schema.sql
docker compose exec db psql -U perekus -d perekus -f /project/seed.sql
docker compose exec db psql -U perekus -d perekus   # консоль для запросов
```

Вариант B — локальный PostgreSQL: `psql -d perekus -f schema.sql` → `-f seed.sql`.

Запросы из `dml.sql` выполняются **выборочно** (Q1–Q7, Q13, Q15 и T1/T2 содержат `:param`-плейсхолдеры — задавать через `psql -v name=value`; Q8–Q12 и Q14 запускаются как есть).
